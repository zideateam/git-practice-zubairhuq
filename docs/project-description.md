# Project Description

## Project Name

Git & GitHub Practical Assignment

## Overview

This project is designed to provide hands-on experience with Git and GitHub, two of the most widely used version control tools in modern software development. The project demonstrates the basic workflow of creating, tracking, and managing source code using Git.

## Project Objectives

The main objectives of this project are:

* Learn how to create and manage a GitHub repository.
* Initialize a local Git repository.
* Create and organize a standard project folder structure.
* Track files using Git.
* Make multiple commits with meaningful commit messages.
* Use a `.gitignore` file to exclude unnecessary files.
* Work with Git branches and merge changes.
* Push local code to a remote GitHub repository.
* Maintain proper project documentation.

## Core Tasks

### 1. Repository Creation

* Create a public GitHub repository.
* Connect the local project with the remote repository.

### 2. Local Project Initialization

* Create the project directory.
* Initialize Git using `git init`.
* Build the required folder structure.

### 3. Basic Program Development

* Create a simple Python program.
* Display the My name ( Zubair Mahmudul Huq).
* Display the current date.

### 4. Version Control

* Add project files to Git.
* Create commits with descriptive messages.
* Track project history.

### 5. Git Ignore Configuration

* Prevent unnecessary files from being tracked.
* Exclude Python cache files and environment files.

*Entry	Purpose
* __pycache__/	Ignore Python cache folder
* *.pyc	Ignore compiled Python files
* .env	Ignore secret environment variables
* venv/	Ignore Python virtual environm

### 6. Branch Management

* Create new branches.
* Develop features independently.
* Merge changes into the main branch.

### 7. Remote Repository Management

* Push local commits to GitHub.
* Keep local and remote repositories synchronized.
